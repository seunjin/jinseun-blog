# 애플리케이션 기획 개요

## 1. 프로젝트 목적
- 개인 블로그 + 포트폴리오 아카이브를 통합한 공간 구축
- 학습 기록, 실험 노트, 프로젝트 사례를 체계적으로 정리하고 공유
- 채용/협업 기회 확대를 위한 전문성 및 성장 스토리 전시

## 2. 핵심 타깃
- 개발자/디자이너 등 기술 커뮤니티 동료
- 잠재 협업자 및 채용 담당자
- 미래의 나(회고/참고용)

## 3. 주요 화면 & 정보 구조
1. **홈(Overview)**
   - 현재 집중하는 일, 대표 프로젝트, 최신 아카이브 항목 3~4개
   - CTA: `Archive`, `About`, 외부 링크(깃허브, 이력서 등)

2. **아카이브(Archive)** — 단일 리스트 기반
   - 전체 콘텐츠를 하나의 피드로 보여주고, 상단 필터/탭으로 1차 분류 전환
   - 2차 태그 필터와 검색을 함께 제공해 미니멀한 탐색 경험 유지
   - 상세 페이지는 라우트별로 제공하지만 상단 네비는 Home/Archive/About 중심 유지

3. **관리자(Admin Workspace)**
   - `/admin` 라우트 그룹으로 분리, 인증된 관리자만 접근
   - 동일한 아카이브 UI에 `isAdmin` 플래그로 수정/삭제/게시 버튼 노출
   - 별도 관리 화면: 글 작성/편집 폼, 카테고리/태그 CRUD, 초안 관리, 통계
   - 관리자 레이아웃에 사이드바(콘텐츠, 카테고리, 설정 등) 제공

4. **소개 & 연락(About & Contact)**
   - 자기소개, 핵심 역량, 사용 기술, 커리어 하이라이트
   - 연락 채널, SNS, 뉴스레터 구독, 오픈소스/발표 링크 등

> 향후 필요 시 Archive 필터 프리셋을 별도 경로(`/archive?category=projects`)로 노출해 독립 페이지처럼 활용 가능

## 4. 콘텐츠 전략 & 분류 체계
- **1차 카테고리(Archive 필터)**
  1. `insights` — 에세이, 회고, 생각 정리
  2. `build-log` — 개발 일지, 실험 노트, 튜토리얼
  3. `projects` — 포트폴리오 사례 연구(Case Study)
  4. `timeline` — 커리어/이벤트 기록, 마일스톤
  5. `resources` *(선택)* — 참고 자료, 툴킷, 북마크
- 카테고리/태그는 관리자에서 CRUD 가능하도록 Supabase 연동 계획 구축
- **2차 태그**: 기술 스택(Next, Supabase 등), 주제(아키텍처, 성능, UI), 상태(draft, in-progress, published) 등을 혼합 사용
- **글 템플릿**: 각 카테고리별 핵심 구성 정의 (예: 프로젝트는 `문제 → 해결 → 결과 → 회고`)
- **프로젝트 업데이트**: 진행 상태(아이디어/진행 중/완료)와 업데이트 로그를 함께 기록
- **미디어 활용**: 코드 스니펫, 다이어그램, 링크 미리보기 등 가독성을 위한 시각 요소 적극 활용
- **다국어 여부**: 우선 한글 중심, 필요 시 주요 글에 영어 요약 추가 고려

## 5. 정보 구조 & 네비게이션
- 글로벌 네비게이션: Home / Blog / Projects / Timeline / About
- 사이드바(선택): 카테고리, 태그, 최신 글, 아카이브
- 푸터: 연락처, SNS, 라이선스, RSS, Newsletter 링크

## 6. 데이터 & 기능 요구사항
- 콘텐츠 저장 방식: Supabase(PostgreSQL) 기반 테이블 설계
  - posts, projects, timeline_events, resources, tags 등
- 분류 관리: `categories`, `tags` 테이블을 두고 관리자에서 CRUD, 빈 데이터 시 기본 프리셋 제공
- 인증/관리: Admin 대시보드 혹은 CMS 연동(향후 Strapi/Keystatic 고려)
- 댓글/피드백: 초기에는 비활성, 이후 외부 서비스(utterances/ giscus) 검토
- 검색: Supabase Full-Text Search 혹은 Algolia 연동 계획 수립

## 7. 단계별 로드맵 (초안)
1. **MVP (Phase 1)**
   - 홈, 블로그, 소개 페이지 정적 콘텐츠
   - Panda 스타일 시스템 정비, 기본 토큰 세팅
   - Supabase 연동 없이 마크다운/MDX 기반으로 시작 가능

2. **Phase 2**
   - Supabase 연동, 게시물/카테고리/태그 CRUD Admin 플로우 구축
   - 프로젝트/타임라인 데이터 구조 도입
   - 태그 필터 및 검색, RSS 생성

3. **Phase 3**
   - E2E 테스트(Playwright) 및 배포 자동화 파이프라인
   - 다국어 지원, 구독(Landing + 이메일) 기능 고려
   - 콘텐트 추천, 관련 글, 아카이브 통계 시각화

## 8. 향후 고려사항
- 브랜딩/디자인 컨셉 확정 후 Panda 토큰 확장 (컬러, 타이포, spacing)
- 퍼포먼스 모니터링 (Vercel Analytics, Supabase 모니터링)
- 접근성(Accessibility) 및 SEO 체크리스트 준비
- 모바일/데스크톱 반응형 전략 문서화
