# 협업 규칙 & 작업 가이드

> 이 문서는 Codex와 함께 작업할 때 지켜야 할 공통 규칙과 워크플로를 정리합니다.

## 커뮤니케이션
- Codex의 모든 작업 결과 및 요약은 기본적으로 **한글**로 제공합니다.
- 필요 시 영어가 더 적합한 용어나 코드 스니펫은 그대로 사용하되, 주변 설명은 한글로 덧붙입니다.

## 작업 기록
- 세션마다 `docs/progress-log.md` 템플릿을 복사해 날짜별로 기록을 남깁니다.
- 로그에는 핵심 작업, 결정 사항, 문제/해결, 다음 할 일을 간결하게 정리합니다.

## 코드 스타일 & 툴링
- 코드 품질과 포매팅은 **Biome**을 기준으로 유지합니다 (`pnpm lint`, `pnpm format`).
- 스타일 시스템은 **Panda CSS**를 사용하며, 필요 시 `pnpm panda:codegen` 또는 `pnpm panda:watch`를 실행해 유틸리티를 동기화합니다.
- Next.js App Router 구조를 기반으로 하며, 글로벌 스타일은 Panda 레이어와 함께 관리합니다.

## 작업 방식
- 변경 사항이 생기면 가능한 한 작은 단위로 커밋합니다.
- 새로운 규칙이나 워크플로 변경이 필요하면 이 문서에 업데이트 내용을 추가하고 로그에도 기록합니다.

## 커밋 메시지 규칙
- 기본 형식은 [Conventional Commits](https://www.conventionalcommits.org/ko/v1.0.0/)를 따릅니다.
- 예시: `feat: 블로그 홈 히어로 섹션 초안 추가`, `chore: panda codegen 스크립트 정리`.
- 긴 변경 사항은 본문에 상세 내용을 적고, 끝에 관련 이슈나 참고 링크를 추가합니다.

## Git 훅 & 자동 검사
- `lefthook`이 `pre-commit` 단계에서 `pnpm lint`를 실행해 품질 검사를 보장합니다.
- 커밋 작성 시 `commitlint`가 메시지를 검사하므로, 규칙을 벗어나면 커밋이 차단됩니다.
- 로컬 훅 설치는 `pnpm install` 혹은 `pnpm prepare` 시 자동으로 수행되며, 수동으로는 `pnpm exec lefthook install`을 실행합니다.

## 테스트 전략
- 1단계: `Vitest`와 `@testing-library/react` 조합으로 컴포넌트/유닛 테스트를 구성합니다.
- 2단계: UI 플로우가 정착되면 `Playwright`로 통합(E2E) 시나리오를 확장합니다.
- 시각 회귀가 필요하면 Playwright의 스냅샷 기능 또는 별도의 도구를 추가 도입합니다.
- 새 테스트 도구를 도입하면 실행 스크립트와 활용법을 README와 이 문서에 함께 기록합니다.

## 향후 확장 아이디어
- 공통 컴포넌트, 디자인 토큰, 번역 등 추가 규칙이 생기면 해당 섹션을 만들어 정리합니다.
- 자동화 스크립트(빌드, 테스트, 배포 등)를 도입할 경우 실행 방법과 주의사항을 명시합니다.
